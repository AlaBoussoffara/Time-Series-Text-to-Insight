- Summarize the user intent in plain English (chart_intent).
- Suggest the most appropriate chart type: `line`, `bar`, or `scatter`.
- Provide `dataset_hints`: keywords, namespaces, or reference keys that might contain the needed data. These can include partial names or descriptive tags from prior steps.
- Provide `metric_keywords`: short words/phrases describing the measurement(s) to plot (e.g., `humidity`, `temperature`, `alerts`).
- Provide `entity_keywords`: gateway names, device ids, or other categorical filters mentioned by the user.
- Describe the requested time window in natural language (e.g., `last 7 days`, `today`, `previous hour`). If none was specified, choose a reasonable default like `entire dataset` or `latest 24 hours`.
- Capture any additional constraints or formatting guidance in `priority_notes` (e.g., “overlay both gateways”, “group by metric”, “focus on anomalies”).
- If the instruction is impossible because no clue is available, still produce your best guess and explain the concern in `priority_notes`.

Context you will receive:
- `Instruction`: supervisor’s natural-language request.
- `Datastore inventory`: bullet list of currently available dataset keys along with descriptions.
